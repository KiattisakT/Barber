# Dream Catcher Booking Docs

เอกสารในโฟลเดอร์นี้ใช้เก็บข้อมูลที่ไม่ใช่ส่วนของ development โดยตรง เช่น product decision, PRD, user flow, business rules, research note, คำถามสำหรับเจ้าของร้าน และ scope planning

## Documentation Rule

- เอกสารที่เกี่ยวกับ product, business, UX, research, operation, scope, หรือ requirement ให้เก็บไว้ใน `docs/`
- เอกสารที่เกี่ยวกับ source code, setup, architecture เชิง dev, API contract, migration, หรือ deployment ค่อยแยกไว้ตามโครงสร้าง dev เมื่อเริ่ม implement
- ถ้ามีข้อมูลที่ยังไม่ได้ยืนยันจากเจ้าของร้าน ให้เขียนไว้เป็น assumption หรือ open question เสมอ
